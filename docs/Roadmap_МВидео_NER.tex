\section{День 0 --- до буткемпа: язык задачи}

Пока нет «вайбкодинга ради скрина». Есть чтение и общий словарь.

\daybox{00}{Теория и насмотренность}{%
\begin{itemize}
  \item Лонгриды / семинары по классическому ML и последовательностям: признаки, overfitting, train/test, метрики.
  \item NLP-база: токенизация, BIO, чем NER отличается от классификации всего запроса целиком.
  \item CRF vs «просто нейросеть»: когда хватает линейной модели на признаках токена.
  \item Word2Vec / эмбеддинги / косинус --- задел под поиск по пространству, не обязательно внедрять в день~1.
  \item Трансформеры --- понимать идею; в MVP не тащить тяжёлый BERT, если SLA $<100$~мс на CPU.
\end{itemize}
}{%
\begin{itemize}
  \item Общий язык: \textit{факт}, \textit{SKU}, \textit{weak label}, \textit{entity F1}, \textit{latency}.
  \item Короткий конспект «что читали / что берём в кейс» (1--2 страницы на команду).
\end{itemize}
}

\noindent\textit{Черновой стек чтения (подставить свои источники):} курсные лонгриды по ИИ, блок про RNN/attention/трансформеры, заметки по TF-IDF и метрикам $P/R/F1$.

% ---------- DAY 3 ----------
\section{День 03 --- Своя разметка и шесть ударов по качеству фактов}

Здесь команда перестаёт «верить словарю» и начинает мерить правду.

\daybox{03}{Gold / synthetic + точность фактов}{%
\begin{itemize}
  \item Ручная разметка выборки запросов (цель: сотни, не «три примера»). Формат: span + тип.
  \item Чуть синтетики: шаблоны \texttt{\{category\} \{brand\} \{attr\}} + шум/опечатки.
  \item Прогнать текущий CRF по gold: честный entity F1 --- это новая правда.
  \item Дожать шесть направлений (ниже) хотя бы на уровне v1.
  \item Дообучить / переобучить CRF (и при необходимости правила) на смеси weak + gold.
\end{itemize}
}{%
\begin{itemize}
  \item Файл gold-разметки в репо + скрипт оценки на нём.
  \item Короткая таблица «было / стало» по ошибкам: бренд$\leftrightarrow$модель, нормализация, ATTR.
\end{itemize}
}

\subsection*{Шесть пунктов --- не wishlist, а backlog дня 3}
\begin{enumerate}[leftmargin=1.4em,itemsep=0.35em]
  \item \textbf{Факты точнее} --- меньше мусора, чтобы фильтры поиска не врали.
  \item \textbf{Бренд $\neq$ модель} --- \textit{iphone}/\textit{galaxy s25} не обязаны быть vendor-строкой один-в-один.
  \item \textbf{Нормализация} --- \textit{айфон}/\textit{iphone}/\textit{Apple} сходятся к канону.
  \item \textbf{Устойчивость к шуму} --- опечатки, регистр, склейки \texttt{16гб}.
  \item \textbf{Своя разметка + синтетика} --- честные метрики, не только self-check словаря.
  \item \textbf{Query$\leftrightarrow$click score} --- правила «согласованной пары» (бренд совпал, токены в title, позиция клика); отбор clean-пар под следующую модель, не «100\% идеал».
\end{enumerate}

% ---------- DAY 4 ----------
\section{День 04 --- Standalone: NER в фундаменте, сверху поиск}

Не заменяем NER эмбеддингами. Склеиваем.

\daybox{04}{Приложение + эмбеддинговый поиск по каталогу}{%
\begin{itemize}
  \item UI/лаба: ввод запроса, подсветка сущностей, JSON, latency vs SLA (уже можно усилить до «поиска»).
  \item Индекс по \texttt{sku\_desc} / сэмплу каталога: эмбеддинги title (TF-IDF+SVD на старте; sentence-embeddings --- если влезаем по времени/CPU).
  \item Ранжирование-гибрид: (a)~фильтры/буст по фактам NER, (b)~косинус в пространстве запрос--SKU.
  \item Показать 5--10 выдач «до/после» фактов --- это лучший слайд защиты.
  \item Заготовка под пункт~6: таблица high-score пар query--click для будущего дообучения.
\end{itemize}
}{%
\begin{itemize}
  \item Standalone-демо: строка поиска $\to$ факты + топ SKU.
  \item Понятная схема на один слайд: \texttt{query $\xrightarrow{\text{NER}}$ facts $\xrightarrow{\text{retrieve}}$ SKU}.
\end{itemize}
}

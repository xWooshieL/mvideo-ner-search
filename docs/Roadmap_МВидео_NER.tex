% ===================== DAY 0 =====================
\section{День 0 --- до буткемпа: просто понять язык}

Пока никто ничего не кодит ради скрина. Читаем и договариваемся, что значат слова.

\daycard{00}{Теория и насмотренность}{%
\begin{enumerate}
  \item Лонгриды и семинары: признаки, overfitting, train/test, метрики --- чтобы потом не путаться.
  \item NLP-база: токены, BIO, чем NER отличается от <<классифицируем весь запрос целиком>>.
  \item CRF vs <<давайте сразу нейросеть>>: когда хватает модели на признаках токена.
  \item Эмбеддинги и косинус --- это задел под поиск. В день 1 тащить не обязательно.
  \item Трансформеры понимать полезно, но тяжёлый BERT в MVP под SLA $<100$ мс на CPU явно не тащим.
\end{enumerate}
}{%
\begin{enumerate}
  \item У всех в голове один словарь: факт, SKU, weak label, entity F1, latency.
  \item Короткий конспект на команду: что прочитали и что реально берём в кейс.
\end{enumerate}
}

\noindent\textit{Что читать:} лонгриды курса, куски про последовательности / attention, TF-IDF и $P/R/F1$. Свои источники --- ок, главное чтобы команда говорила на одном языке.

% ===================== DAY 2 =====================
\section{День 02 --- Сюда нельзя опоздать: EDA + MVP}

Если к вечеру нет JSON-сервиса --- на день 3 не прыгаем в <<приложение мечты>>. Сначала дожимаем факты.

\daycard{02}{EDA, словари, CRF, тонкий сервис}{%
\begin{enumerate}
  \item EDA: длины запросов, топы, цена vs позиция, heatmaps --- пачка картинок в figures/.
  \item Из кликов собрать словари брендов и категорий $\to$ artifacts/*.txt.
  \item Правила на ATTR (память, цвет, диагональ...) + авто-BIO.
  \item Обучить CRF: по сути максимизируем $P(y\mid x)$, где $y$ --- теги, $x$ --- слова запроса.
  \item Поднять /extract (и debug), чтобы любой из команды вбил запрос и сразу увидел JSON.
\end{enumerate}
}{%
\begin{enumerate}
  \item EDA закрыт: ноутбуки + куча графиков.
  \item MVP-ядро живо: запрос $\to$ факты, latency нормально укладывается в SLA.
  \item В доке честно написано: метрики пока против weak labels, не против ручного gold.
\end{enumerate}
}

% ===================== DAY 3 =====================
\section{День 03 --- Своя разметка и шесть ударов по качеству}

Тут перестаём слепо верить словарю и начинаем мерить правду.

\daycard{03}{Gold / синтетика / переобучение}{%
\begin{enumerate}
  \item Руками разметить пачку запросов (лучше сотни, не три примера). Span + тип.
  \item Чуть синтетики: шаблоны <<категория бренд attr>> + шум и опечатки.
  \item Прогнать текущий CRF по gold --- вот это уже честный F1.
  \item Закрыть шесть пунктов ниже хотя бы в версии v1.
  \item Переобучить модель на смеси weak + gold.
\end{enumerate}
}{%
\begin{enumerate}
  \item Gold-файл в репо + скрипт оценки на нём.
  \item Табличка <<было / стало>> по типичным косякам.
\end{enumerate}
}

\subsection*{Шесть пунктов --- это backlog, а не мечты в стол}
\begin{enumerate}
  \item Факты точнее --- меньше мусора, фильтры поиска не врут.
  \item Бренд и модель не путаем: iphone / galaxy s25 --- это не всегда просто vendor-строка.
  \item Нормализация: айфон / iphone / Apple сходятся к одному канону.
  \item Устойчивость к шуму: опечатки, регистр, склейки вроде 16гб.
  \item Своя разметка + синтетика --- честные метрики, а не self-check словаря.
  \item Score пары query$\leftrightarrow$click: бренд совпал, токены в title, позиция клика. Берём <<хорошо согласованные>> пары, без сказки про 100\% идеал.
\end{enumerate}

% ===================== DAY 4 =====================
\section{День 04 --- Standalone: NER внизу, поиск сверху}

Эмбеддинги не вместо NER. Вместе.

\daycard{04}{Приложение + поиск по каталогу}{%
\begin{enumerate}
  \item UI: строка поиска, подсветка сущностей, JSON, latency vs 100 мс.
  \item Индекс по sku\_desc / куску каталога. На старте можно TF-IDF+SVD, если sentence-embeddings не влезают по времени.
  \item Гибрид: буст/фильтры по фактам NER + близость запрос--SKU в пространстве.
  \item Сделать 5--10 выдач <<до / после фактов>> --- это лучший слайд на защите.
  \item Заготовка под пункт 6: таблица high-score пар query--click.
\end{enumerate}
}{%
\begin{enumerate}
  \item Демо: вбил запрос $\to$ увидел факты и топ SKU.
  \item Одна схема на слайд: query $\to$ NER $\to$ facts $\to$ retrieve $\to$ SKU.
\end{enumerate}
}
